# 2D 精灵表格导出

一个面向 `macOS` 的 2D 精灵抠图、切图和精灵表导出工具。

当前主流程是：导入图片，用类似 PS 魔棒的“边缘连通背景选择”算法把背景变透明，在透明结果上选择图案，然后按“切图导出”或“精灵表导出”两种模式输出 PNG。

## 当前主流程

- 导入 `PNG / JPG / WEBP`
- 自动从图片四边取样背景色
- 聚类出 1-3 个主要背景色
- 从图片边缘种子开始 flood fill，只移除与边缘连通的背景
- 对紧邻背景的半背景边缘像素降低 alpha，减少白边
- 可把可见图案外轮廓向内缩 0-4 像素，用来剪掉顽固白边
- 预览原图和透明 PNG 结果
- 导出 `cutout_<原文件名>.png`
- 导出尺寸保持和原图一致
- 在透明结果上自动识别分离图案，支持点选框选
- 将选中的图案逐个导出为独立透明 PNG
- 将选中的图案按 `32x32 / 64x64 / 自定义` 单元格导出为单张精灵表
- 顶部只保留 `导入图片 / 切图导出 / 精灵表导出` 三个入口
- 桌面宽屏下左侧预览固定，右侧配置栏独立滚动

## 为什么换成魔棒式算法

之前的精灵切分路线会先把整张图拆成多个连通块，再在每个块里做边缘净化。这个方案适合拼精灵表，但对白衣服、白床单、浅色桌面这类素材容易遇到冲突：背景容差调大能少白边，但也可能打穿主体。

魔棒式算法把职责拆开：

- 背景只从图片边缘开始选，主体内部相近颜色不会被删除。
- 处理顺序是“边缘连通背景透明化 -> 边缘内缩 -> 边缘柔化”，柔化会作用在内缩后新产生的透明边界上。
- 第一版不做目标尺寸重采样，也不拼精灵表，先集中解决抠图质量。

## 参数说明

- `容差`：背景颜色允许的差异。背景是纯白时可以低一些，近白或轻微压缩杂色时可以调高。
- `边缘柔化`：降低紧贴背景的半背景像素 alpha，用来缓解白边。
- `边缘内缩`：对透明区域旁边的前景像素做 alpha 腐蚀。默认 1 像素，适合剪掉最外圈白边；如果细线被削掉，可以调回 0。
- `边缘采样宽度`：从图片边缘向内采样多少像素来估计背景。
- `背景色数量`：最多聚类几个背景色，默认 3，适合边缘存在轻微色差的图片。
- `精灵表单元格`：默认 `32x32`，也可选 `64x64` 或自定义宽高。
- `固定列数`：默认 4 列，行数自动计算。
- `采样模式`：`像素` 使用最近邻，`平滑` 使用双三次；两者都保持比例并居中透明留白。

## 适合的素材

当前最适合：

- 白底或近白底像素素材
- 纯色背景角色/物件图
- 背景与图片边缘连通的素材
- 黑描边像素角色，主体内部有白衣服或浅色物体

当前不适合：

- 复杂摄影背景
- 背景不连到图片边缘的图
- 需要手动点击多处取样的素材
- 需要语义级 AI 抠图的插画

## 本地开发

### 环境

- Node.js 24+
- npm 11+

### 安装依赖

```bash
npm install
```

### 启动开发模式

```bash
npm run dev
```

### 运行测试

```bash
npm test
```

### 构建

```bash
npm run build
```

### 打包 macOS 应用

```bash
npm run package
```

打包产物会出现在 `release/` 目录。本阶段默认不做签名、公证和 DMG。

## 项目结构

```text
src/
  core/
    magicWand.ts   魔棒式边缘连通抠图核心
    cutoutSprites.ts  基于透明 alpha 的连通域切图
    cutoutSpritesheet.ts  单组切图精灵表导出
    detection.ts   旧多精灵检测逻辑，暂时保留
    export.ts      旧精灵表导出逻辑，暂时保留
    types.ts       共享图像与业务类型
  main/            Electron 主进程
  preload/         Electron bridge
  renderer/        React 界面、参数和预览
  types/           Electron 类型
tests/
  core.test.ts
```
